# Claude Code Guide

## Best Uses
- Break product ideas into implementation plans.
- Review architecture and tradeoffs.
- Draft detailed documentation.
- Review Codex-generated code at a higher level.
- Explain unfamiliar technologies.

## Prompt Pattern
```text
Context:
Decision needed:
Current constraints:
Options considered:
Output format:
```

## Expectations
- Prefer practical solo-developer workflows.
- Keep architecture appropriate for an MVP.
- Identify risks and missing acceptance criteria.
- Help turn broad ideas into Jira-ready tasks.

## Review Focus
- Domain model clarity.
- API consistency.
- Overengineering risks.
- Missing tests or missing study notes.
